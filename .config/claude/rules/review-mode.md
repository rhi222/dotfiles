# レビューモード

PRレビュー時は以下の観点を優先する。

## レビュー観点（優先度順）

1. **セキュリティ**: SQLインジェクション、XSS、ハードコードされたシークレット、認証・認可の不備
2. **正確性**: ロジックの誤り、エッジケース、エラーハンドリングの不足
3. **テスト網羅性**: 変更に対するテストの有無、境界値テスト
4. **パフォーマンス**: N+1クエリ、不要な再レンダリング、メモリリーク
5. **可読性**: 命名、関数の責務分離、過度な複雑性

## レビューコメントの分類

- **must**: 修正必須（セキュリティ、バグ）
- **imo**: 個人的な改善提案
- **nits**: 軽微なスタイル指摘
- **q**: 意図の確認
