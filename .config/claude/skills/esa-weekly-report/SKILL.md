# esa 週次エグゼクティブレポート生成

複数のesa記事の直近一週間の差分を収集し、部長会向けエグゼクティブレポートを生成する。

追加の記事指定（任意）: $ARGUMENTS
形式: `<team> <post_number> [<team> <post_number> ...]`

## 参照スキル

- `.config/claude/skills/esa-api/SKILL.md` — esa API アクセス方法

## 手順

### 1. 対象記事リストの構築

1. `.config/claude/skills/esa-weekly-report/esa-weekly-report-posts.json` から固定リストを読み込む
2. `$ARGUMENTS` に追加の `<team> <post_number>` ペアがあればリストに追加

### 2. 各記事の差分収集

対象記事ごとに以下を実行する。各記事は独立しているため並列Bash呼び出しで処理すること。

1. `esa-api` スキルに従い、リビジョン一覧を取得
2. 直近7日間のリビジョンをフィルタ（`created_at` で判定）
3. compare URL を生成: `https://<team>.esa.io/posts/<number>/revisions/compare/<from>...<to>/html_diff`
4. compare API で diff 内容を取得

**差分が無い場合:**
- 直近7日間にリビジョンがない → 「変更なし」と記録し、レポートでは省略
- リビジョンが1件のみ（FROM=TO） → 全リビジョン一覧から1つ前のリビジョン番号を取得し比較

### 3. エグゼクティブレポート生成

収集した全記事の差分を統合し、以下の構造でレポートを作成する:

```markdown
# 週次レポート（YYYY-MM-DD）

## サマリ
- 【検索】変更概要（1行）
- 【素材登録】変更概要（1行）
- 【TACO】変更概要（1行）
- 【予約】変更概要（1行）
- 【基盤 / システム名】変更概要（1行）
- ...
※ サブシステム名は記事の label を使用
※ 基盤のみ、記事内のシステム名（航空キュー基盤、料金計算基盤 等）の粒度で分けて記載
※ リスクや特に注意が必要な事項があれば ⚠ を付けて明示

## 記事別の変更点

### [label]
- compare URL: <url>
- 主な変更点
  - ...

### [label]
...
```

レポートのルール:
- 記事別の変更点: diff から読み取れる事実のみ記載
- サマリ: 事実に基づく仮説や示唆を含めてよいが、仮説部分は「（仮説）」と明示する
- 技術的詳細は最小限に抑える
- 出力後にユーザーがレビューするため、仮説の正誤はユーザー判断に委ねる

### 4. 出力

1. **ファイル保存**: `~/Obsidian/05_Organization/Buchokai/YYYY-MM-DD.md` に保存
   - ファイル名の日付は今日の日付
   - 既存ファイルがある場合はユーザーに確認
2. **会話内表示**: レポートのサマリを会話内にも表示
