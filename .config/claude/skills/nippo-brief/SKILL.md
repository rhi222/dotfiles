# 日報サマリー表示

> 日報の作業ログから事実のみを箇条書きで要約し、端的なサマリーを表示する

## 概要

nippo-show が日報の全文表示であるのに対し、nippo-brief は要約表示。作業ログから重要な事実のみを抽出して3〜5項目の箇条書きにまとめる。

## 入力

| 項目     | パス                                        | 説明             |
| -------- | ------------------------------------------- | ---------------- |
| **入力** | `~/Obsidian/02_Daily/nippo.YYYY-MM-DD.md`   | 日報ファイル     |

## 実行スクリプト

```bash
TARGET_DATE="${ARGUMENTS:-$(date +%Y-%m-%d)}"
NIPPO_FILE="$HOME/Obsidian/02_Daily/nippo.${TARGET_DATE}.md"

echo "📋 日報サマリー - ${TARGET_DATE}"
echo "================================"

if [ ! -f "$NIPPO_FILE" ]; then
    echo "❌ 日報ファイルが見つかりません: $NIPPO_FILE"
    echo "まず /nippo-add でタスクを記録してください。"
    exit 1
fi

cat "$NIPPO_FILE"
```

## 出力フォーマット

以下の形式で端的なサマリーを表示してください：

```markdown
## 📋 日報サマリー - [日付]

### やったこと（3〜5項目）
- [作業ログから抽出した事実1]
- [作業ログから抽出した事実2]
- [作業ログから抽出した事実3]

### ⏱ 時間サマリ
| タスク | 時間 |
| ------ | ---- |
| {タスク名} | {経過時間} |
| **計測済み合計** | **{合計}** |

### 📊 ステータス
- 完了タスク: {n}件
- 未完了タスク: {n}件
- 作業ログエントリ: {n}件
```

## 注意事項

- 事実のみを抽出する（解釈や評価は加えない）
- 箇条書きは3〜5項目に絞る（多すぎると要約の意味がない）
- 時間サマリはstart/endログが記録されている場合のみ表示
- 日報ファイルに追記はしない（表示のみ）
