# 問いかけフレームワーク

## 理論的背景

### コルブの経験学習サイクル

1. **具体的経験** (Concrete Experience): 何が起きたか
2. **内省的観察** (Reflective Observation): 何を感じ、何に気づいたか
3. **抽象的概念化** (Abstract Conceptualization): どんな教訓・原則を導けるか
4. **能動的実験** (Active Experimentation): 次にどう活かすか

### ギブスのリフレクティブサイクル

1. **記述** (Description): 何が起きたか
2. **感情** (Feelings): 何を感じたか
3. **評価** (Evaluation): 何が良くて何が悪かったか
4. **分析** (Analysis): なぜそうなったか
5. **結論** (Conclusion): 他に何ができたか
6. **行動計画** (Action Plan): 次にどうするか

## カテゴリ別の問いパターン

### 1. 判断振り返り

作業ログから判断・意思決定のポイントを見つけ、以下のパターンで問いを作る：

- 「[具体的行動]を選んだ時、他にどんな選択肢がありましたか？その中でなぜこれを選びましたか？」
- 「[具体的場面]で、判断の根拠となった情報は十分でしたか？不足していた情報は何ですか？」
- 「[具体的判断]を、もう一度同じ状況で行うとしたら、同じ判断をしますか？」

### 2. 感情記録

作業ログから感情が動いた（であろう）ポイントを見つけ、以下のパターンで問いを作る：

- 「[具体的場面]の時、どんな気持ちでしたか？その感情は作業にどう影響しましたか？」
- 「今日最も集中できた瞬間はいつですか？その時何が集中を助けていましたか？」
- 「[具体的な困難]に直面した時、最初に何を感じましたか？」

### 3. 代替案探索

作業ログの具体的アプローチに対して、別の可能性を探る：

- 「[具体的アプローチ]の代わりに、まったく違うアプローチを取るとしたら？」
- 「[具体的な問題解決]について、[別の立場の人]ならどうアプローチしたと思いますか？」
- 「もし時間の制約がなかったら、[具体的タスク]にどう取り組みたかったですか？」

### 4. 学び言語化

作業ログから学びの種を見つけ、言語化を促す：

- 「[具体的な経験]から、今後も使えそうな原則や教訓はありますか？」
- 「今日の[具体的作業]を通じて、自分の中で何が変わりましたか？」
- 「[具体的発見]について、1ヶ月前の自分に教えるとしたら、どう説明しますか？」

### 5. 明日の意思決定

翌日の行動に繋げる問い：

- 「今日の経験を踏まえて、明日最初に取り組むべきことは何ですか？その理由は？」
- 「[今日の未完了タスク]について、明日はどんなアプローチで臨みますか？」
- 「今日学んだことを、明日の[具体的場面]でどう活かせますか？」

## 問い生成のルール

1. **必ず作業ログの具体的エピソードに言及する**（汎用的な問いは禁止）
2. **1カテゴリ1〜2問**（合計5〜7問）
3. **open-ended**（Yes/Noで答えられない形式）
4. **非断定的**（「〜すべき」ではなく「〜について、どう感じましたか？」）
5. **回答は絶対に生成しない**（回答欄は空白）
