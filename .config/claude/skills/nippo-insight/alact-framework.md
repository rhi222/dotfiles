# ALACTモデル フレームワーク

## ALACTモデルとは

Fred Korthagen が提唱した、教師教育における振り返りモデル。5つの段階を通じて、経験から深い学びを引き出す。

## 5段階

### 1. Action（行為）

**目的**: 振り返りの対象となる行為を特定・記述する

**問いの例**:

- 「具体的に何をしましたか？」
- 「その時の状況はどのようなものでしたか？」
- 「その行為に至る直前、何がきっかけでしたか？」

**AIの役割**: 作業ログから対象の行為を抽出し、客観的に記述する

### 2. Looking back on the action（行為の振り返り）

**目的**: 行為の中で起きたこと・感じたことを振り返る

**問いの例**:

- 「その時、何を考えていましたか？」
- 「どんな感情が湧きましたか？」
- 「相手（チームメンバー、コード、システム）はどう反応しましたか？」
- 「予想と違ったことはありましたか？」

**AIの役割**: 問いを生成し、関連する心理学的概念（認知バイアスなど）を参考情報として付記

### 3. Awareness of essential aspects（本質的な気づき）

**目的**: 行為の背後にある本質的なパターンや要因に気づく

**問いの例**:

- 「この経験と過去の似た経験に共通するパターンはありますか？」
- 「なぜこのアプローチを選んだのか、深層の理由は何ですか？」
- 「この行為が示しているあなたの傾向や価値観は何ですか？」

**AIの役割**: 問いを生成し、関連するソフトウェア工学の原則やメンタルモデルを参考情報として付記

### 4. Creating alternative methods of action（代替行動の創造）

**目的**: 次に同じ状況になった時の別のアプローチを考える

**問いの例**:

- 「次に同じ状況になったら、どんな別のアプローチが考えられますか？」
- 「理想的にはどうしたかったですか？」
- 「[別の視点の人]ならどうアプローチしたと思いますか？」

**AIの役割**: 問いを生成し、関連する技術的パターンやベストプラクティスを参考情報として付記

### 5. Trial（試行）

**目的**: 新しい行動仮説を立て、次の機会に試す

**問いの例**:

- 「次に試したいアプローチは何ですか？」
- 「そのアプローチを試す最初のステップは何ですか？」
- 「成功をどう測りますか？」

**AIの役割**: 着手候補を提示するが、**最終的な行動仮説の決定はユーザーが行う**

## 振り返り対象の選定基準

1. **判断を伴った行為**: 最も学習価値が高い
2. **試行錯誤した行為**: 失敗からの学びが豊富
3. **新しい挑戦**: 成長の機会
4. **感情が動いた行為**: 深い振り返りにつながりやすい
5. **チームに影響を与えた行為**: 組織学習の種
