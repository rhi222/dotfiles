文章からAI特有のパターンを検出・修正し、人間が書いたような自然な文章に変換するスキルです。

## 使い方

### ケース1: ユーザーが文章を渡す場合

ユーザーが「この文章をhumanizeして」と文章を渡してきた場合、その文章に対して2パス処理を実行します。

### ケース2: 自己修正

Claudeが生成した文章に対して「AIっぽいのを直して」と言われた場合、直前の出力に対して2パス処理を実行します。

## 処理フロー（2パス）

### Pass 1: パターン検出

文章を修正せず、以下のパターン一覧に基づいて問題箇所を特定します。

- 該当パターンのIDと箇所を列挙する
- 検出されなかったカテゴリは省略する

### Pass 2: 修正実施

検出結果に基づいて文章を修正し、出力フォーマットに従って結果を返します。

- 修正は最小限にとどめ、元の意味を変えない
- 文章全体のトーンと文脈を考慮する
- 一度に全パターンを直すのではなく、不自然さが大きいものから優先する

## AIくさいパターン一覧

### A. コンテンツ系

| ID | パターン | 検出対象 |
|----|---------|---------|
| A1 | 重要性の誇張 | 「革命的」「画期的」「注目に値する」「非常に重要」 |
| A2 | 曖昧な出典 | 「研究によれば」「専門家は指摘しています」「多くの人が」 |
| A3 | 定型的な結論 | 「今後の展開が注目されます」「さらなる発展が期待されます」「〜していくことが重要です」 |
| A4 | マーケティング調 | 「優れた」「高度な」「包括的な」「洗練された」「シームレスな」 |

### B. 言語系

| ID | パターン | 検出対象 |
|----|---------|---------|
| B1 | AI頻出語彙 | 段落冒頭の「さらに」「加えて」「また」「それに加えて」の連続使用 |
| B2 | copula回避 | 「〜の役割を果たしています」「〜として位置づけられています」「〜を可能にします」 |
| B3 | ing付け足し | 「〜を浮き彫りにしており」「〜を示唆しています」「〜を体現しています」 |
| B4 | 3要素列挙 | 「まず〜次に〜最後に〜」「第一に〜第二に〜第三に〜」の機械的パターン |
| B5 | 過剰ヘッジング | 「〜と言えるでしょう」「〜かもしれません」「〜の可能性があります」の連続 |

### C. スタイル系

| ID | パターン | 検出対象 |
|----|---------|---------|
| C1 | ダッシュ過用 | em dash「——」、全角ダッシュ「―」の多用 |
| C2 | 太字過用 | 「**見出し:** 内容」形式のインラインヘッダーの乱用 |
| C3 | 箇条書き依存 | 文章で書けるのに機械的に箇条書きで羅列しているケース |
| C4 | 絵文字過用 | 技術文書・解説文中の不要な絵文字使用 |

### D. コミュニケーション系

| ID | パターン | 検出対象 |
|----|---------|---------|
| D1 | チャットボット残留 | 「参考になれば幸いです」「以上が〜の概要です」「ご不明な点があれば」 |
| D2 | 追従的トーン | 「素晴らしいご質問ですね」「鋭いご指摘です」「とても良い視点です」 |
| D3 | 学習データ免責 | 「私の学習データの範囲では」「最新の情報は異なる可能性があります」 |

具体的なBefore/After例が必要な場合は `references/ja-patterns.md` を参照してください。

## 「魂のある文章」への転換ガイド

パターン除去だけでは平板な文章になる。以下の観点で能動的に改善する。

### 意見を持つ

曖昧なヘッジを削り、筆者の立場を明確にする。「〜と言えるかもしれません」→「〜だ」。ただし根拠がない断言は避ける。

### リズムを崩す

短文と長文を混ぜる。すべて同じ長さの文が並ぶとAI感が増す。体言止めや倒置も有効。

### 具体的に書く

「さまざまな」「多くの」を具体例に置き換える。「さまざまなメリットがあります」→「読み込み速度が3倍になる」。

## 出力フォーマット

```
[修正後の文章をここに出力]

---
**修正サマリー**
- [パターンID] 修正内容の簡潔な説明
- [パターンID] 修正内容の簡潔な説明
```

例:
```
---
**修正サマリー**
- [A1] 「革命的な変化」→「大きな変化」に変更
- [B1] 段落冒頭の「さらに」「加えて」の連続を解消
- [D1] 「参考になれば幸いです」を削除
```

## 注意事項

- **文脈判断**: ビジネスメールの敬語（「何卒よろしくお願いいたします」等）は日本語の文化的慣習であり、AIくささではない。修正対象外とする
- **技術的正確性**: 修正によって技術的な正確性が損なわれないようにする
- **意味の保持**: 元の文章の主張・意図を変えない。表現を変えるだけにとどめる
- **過剰修正の防止**: すべてのパターンを機械的に除去するのではなく、文脈に応じて判断する。「さらに」が1回だけ使われている場合は問題ない
- **文体の一貫性**: 修正後の文章内でです・ます調と、だ・である調が混在しないようにする
