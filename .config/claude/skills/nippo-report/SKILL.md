# 進捗報告フォーマット生成

> 日報の作業ログを上司・メンターが1分で読める進捗報告フォーマットに変換する

## 概要

日報の作業ログを簡潔な進捗報告フォーマットに変換する。上司やメンターへの報告用テキストとして使用できる形式で出力する。

## 入力・出力

| 項目     | パス                                        | 説明             |
| -------- | ------------------------------------------- | ---------------- |
| **入力** | `~/Obsidian/02_Daily/nippo.YYYY-MM-DD.md`   | 日報ファイル     |
| **出力** | 標準出力（日報には追記しない）               | 進捗報告テキスト |

## 実行スクリプト

```bash
TARGET_DATE="${ARGUMENTS:-$(date +%Y-%m-%d)}"
NIPPO_FILE="$HOME/Obsidian/02_Daily/nippo.${TARGET_DATE}.md"

echo "📊 進捗報告生成 - ${TARGET_DATE}"
echo "================================"

if [ ! -f "$NIPPO_FILE" ]; then
    echo "❌ 日報ファイルが見つかりません: $NIPPO_FILE"
    echo "まず /nippo-add でタスクを記録してください。"
    exit 1
fi

echo "📖 日報内容:"
cat "$NIPPO_FILE"
echo ""
echo "✅ データ読み込み完了"
```

## 出力フォーマット

以下の進捗報告フォーマットで出力してください。日報には追記せず、標準出力のみ。
ユーザーがコピー＆ペーストしやすいよう、コードブロックで囲んで表示してください。

```markdown
## 進捗報告 - [日付]

### 今日の成果
- [完了した主要な作業1]
- [完了した主要な作業2]

### 進行中の課題
- [取り組み中の課題とその進捗]

### 支援が必要な点
- [ブロッカーや相談事項があれば記載、なければ「特になし」]

### 明日の予定
- [翌日に予定している作業]
```

## 注意事項

- 事実ベース・簡潔に（上司が1分で読めるレベル）
- 技術的な詳細は省略し、成果と課題にフォーカス
- 日報ファイルには追記しない（表示のみ）
- 出力はコードブロックで囲み、コピーしやすくする
