# 進捗報告 出力フォーマット

## フォーマット要件

- **簡潔さ**: 上司が1分で読めるレベル
- **事実ベース**: 推測や感想は含めない
- **構造化**: 4セクション固定

## 出力テンプレート

以下のフォーマットで出力し、コードブロックで囲んでコピーしやすくする：

```markdown
## 進捗報告 - [日付]

### 今日の成果

- [完了した主要な作業。具体的な成果物があれば明記]
- [2〜4項目程度]

### 進行中の課題

- [課題名]: [現在の進捗と次のステップ]
- [ブロッカーがあれば明記]

### 支援が必要な点

- [具体的な支援内容。なければ「特になし」]

### 明日の予定

- [予定している主要な作業]
- [1〜3項目程度]
```

## 変換ルール

1. 作業ログの時系列エントリ → 成果・課題に再構成
2. 未完了タスク → 「進行中の課題」または「明日の予定」
3. 完了タスク → 「今日の成果」
4. ブロッカーや相談メモ → 「支援が必要な点」
5. 次の一手/着手候補 → 「明日の予定」
