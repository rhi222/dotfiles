# PR監視ループ

PRがマージ可能になるまで、レビューコメント対応とCI修正を自動で繰り返す。

## 手順

1. **PR特定**: `$ARGUMENTS` にPR番号があればそれを使う。なければ
   `gh pr view --json number,url,title` で現在のブランチのPRを特定する。
   PRが見つからなければその旨を報告して終了する。
2. **開始確認**: 対象PR（番号・タイトル・URL）と停止条件を表示し、
   ユーザーに開始してよいか確認する（自動push を伴うため必須）。
3. **ループ起動**: `/loop` スキルを間隔 `10m`、以下のプロンプトで起動する。

   > PR #<番号> の状態を確認する。
   > (1) 未対応のレビューコメントがあれば pr-feedback スキルの手順に従って
   >     コードを修正しpushする。コメントへの返信投稿は行わない。
   > (2) CIチェックが失敗していれば `gh pr checks <番号>` で失敗ジョブを特定し、
   >     ログを調査して修正・pushする。
   > (3) 全CIチェックがpassし、かつ未対応のレビューコメントがゼロなら、
   >     対応内容のサマリを報告してループを停止する。

## 注意

- コミットメッセージに `Claude-Session: https://...` などのセッションURLトレーラーを含めない
- コンフリクトが発生した場合は自力rebaseせず、状況を報告してループを停止する
- 同一の失敗が3回連続した場合は無限ループとみなし、状況を報告して停止する
