# gcal-workspace-export

会社の Google Workspace カレンダーに登録された勤務場所（Working Location）を、個人の Google カレンダーへ自動連携するための Google Apps Script（GAS）ツール群。

## 概要

2つのスクリプトで構成される：

```
会社 GCal (勤務場所)
    ↓  work-app.gs（会社の Workspace で実行）
会社 Spreadsheet (work_events シート)
    ↓  手動 or コピーで個人 Spreadsheet へ転記
個人 Spreadsheet (work_events シート)
    ↓  private-app.gs（個人アカウントで実行）
個人 GCal (終日イベントとして登録)
```

### work-app.gs — 勤務場所エクスポート

会社の Google Workspace 環境で実行する GAS。プライマリカレンダーから `workingLocation` イベントを取得し、Spreadsheet の `work_events` シートへ書き出す。

**機能（スプレッドシートメニュー）:**

| メニュー項目                         | 説明                                   |
| ------------------------------------ | -------------------------------------- |
| 勤務場所を同期（過去14日〜未来60日） | 過去の変更巻き取りを含むフル同期       |
| 勤務場所を同期（未来だけ）           | 今日以降のみ同期                       |
| 勤務場所を同期（更新のみ）           | 既存行の更新のみ（新規行は追加しない） |

**出力カラム:**

| カラム      | 内容                 |
| ----------- | -------------------- |
| `work_date` | 日付（YYYY-MM-DD）   |
| `work_type` | 在宅 / 出社 / 外出   |
| `office`    | オフィス名（出社時） |
| `memo`      | 備考                 |

**必要な API:**

- Calendar API（Advanced Service）を有効化する必要がある

### private-app.gs — 個人カレンダーへの登録

個人の Google アカウントで実行する GAS。Spreadsheet の `work_events` シートの内容を個人カレンダーに終日イベントとして登録する。

**機能（スプレッドシートメニュー）:**

| メニュー項目                 | 説明                                 |
| ---------------------------- | ------------------------------------ |
| 未登録を個人カレンダーへ登録 | `imported=FALSE` の行のみ新規登録    |
| 上書き同期                   | 既存イベントを更新、無ければ新規作成 |
| ドライラン                   | 登録せず対象件数を確認               |

**入力カラム（A〜D は work-app.gs と同じ、E〜G は本スクリプトが管理）:**

| カラム              | 内容                         |
| ------------------- | ---------------------------- |
| `imported` (E)      | 登録済みフラグ（TRUE/FALSE） |
| `imported_at` (F)   | 登録日時                     |
| `gcal_event_id` (G) | 作成されたイベントの ID      |

**イベントタイトルの装飾:**

| work_type | タイトル       |
| --------- | -------------- |
| 出社      | 🏢 諒平出社    |
| 在宅      | 🏠 諒平在宅    |
| 有休      | 🌿 諒平有休    |
| その他    | 📍 {work_type} |

## セットアップ

### work-app.gs（会社側）

1. 会社の Google アカウントで新規スプレッドシートを作成
2. 「拡張機能」→「Apps Script」を開く
3. `work-app.gs` の内容を貼り付け
4. Apps Script エディタで「サービス」から **Calendar API (v3)** を追加
5. スプレッドシートをリロードするとメニューに「Working Location Sync」が表示される

### private-app.gs（個人側）

1. 個人の Google アカウントで新規スプレッドシートを作成
2. `work_events` シートに会社側と同じヘッダー（A〜G列）を用意
3. 「拡張機能」→「Apps Script」を開く
4. `private-app.gs` の内容を貼り付け
5. `getTargetCalendar_()` のカレンダー ID を自分の環境に合わせて変更
6. スプレッドシートをリロードするとメニューに「Work Sync」が表示される

## 運用フロー

1. 会社のスプレッドシートで「勤務場所を同期」を実行
2. 出力された `work_events` の内容を個人のスプレッドシートへ転記（手動コピー）
3. 個人のスプレッドシートで「未登録を個人カレンダーへ登録」を実行
4. 変更があった場合は「上書き同期」で既存イベントを更新
